# Feature 005: Patient Management

## Spec

### Overview

Patient Management enables clinics to maintain a database of patients with contact information, appointment history, notes, and CRM tags. Patients can belong to multiple clinics and can be searched by name, phone, or CPF.

---

### User Stories

#### US-001: Add Patient

**As a** clinic staff
**I want to** add a new patient
**So that** I can book appointments for them

**Acceptance Criteria:**
- [ ] Name (required)
- [ ] Phone (required)
- [ ] Email (optional)
- [ ] Date of birth (optional)
- [ ] CPF (optional)
- [ ] Address (optional)

#### US-002: Edit Patient

**As a** clinic staff
**I want to** edit patient information
**So that** I can keep records updated

**Acceptance Criteria:**
- [ ] All fields editable
- [ ] Save confirmation

#### US-003: Search Patients

**As a** clinic staff
**I want to** search for patients
**So that** I can find them quickly

**Acceptance Criteria:**
- [ ] Search by name
- [ ] Search by phone
- [ ] Search by CPF
- [ ] Real-time results

#### US-004: View Appointment History

**As a** clinic staff
**I want to** see a patient's appointment history
**So that** I understand their visits

**Acceptance Criteria:**
- [ ] List of all appointments
- [ ] Show date, professional, service
- [ ] Show status
- [ ] Sorted by date (recent first)

#### US-005: Filter by Professional

**As a** clinic staff
**I want to** filter patients by professional
**So that** I see a specific doctor's patients

**Acceptance Criteria:**
- [ ] Dropdown filter
- [ ] Show patients who had appointments with professional
- [ ] Clear filter option

**As a** clinic staff
**I want to** add notes to patient records
**So that** I remember important information

**Acceptance Criteria:**
- [ ] Free-text notes field
- [ ] Notes persist
- [ ] Visible on patient detail

#### US-007: CRM Tags

**As a** clinic staff
**I want to** tag patients
**So that** I can categorize them

**Acceptance Criteria:**
- [ ] Add multiple tags
- [ ] Remove tags

#### FR-002: Patient Search Implementation

```typescript
async function searchPatients(
  clinicId: string,
  query: string
): Promise<Patient[]> {
  // Clean and normalize query
  const normalizedQuery = query.toLowerCase().trim();
  const phoneQuery = query.replace(/\D/g, '');  // Remove non-digits for phone

  // Query Firestore
  const patients = await db.collection('gendei_patients')
    .where('clinicIds', 'array-contains', clinicId)
    .get();

  // Filter in-memory (Firestore doesn't support OR queries well)
  return patients.docs
    .map(doc => doc.data() as Patient)
    .filter(patient => {
      const nameMatch = patient.name.toLowerCase().includes(normalizedQuery);
      const phoneMatch = patient.phone.includes(phoneQuery);
      const cpfMatch = patient.cpf?.replace(/\D/g, '').includes(phoneQuery);
      return nameMatch || phoneMatch || cpfMatch;
    });
}
```

#### FR-003: Patient Statistics Update

```typescript
async function updatePatientStats(patientId: string): Promise<void> {
  const appointments = await db.collectionGroup('appointments')
    .where('patientId', '==', patientId)
    .where('status', 'not-in', ['cancelled'])
    .get();

  const stats = {
    totalAppointments: appointments.size,
    lastAppointmentAt: appointments.docs
      .map(doc => doc.data().createdAt)
      .sort((a, b) => b - a)[0] || null,
  };

  await db.collection('gendei_patients').doc(patientId).update(stats);
}
```
